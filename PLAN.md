# Gym App — Project Plan

## Overview
iOS-only AI-native gym app. The AI is the coach — it programs your training, adapts daily based on your data, and runs the entire workout experience. Users open the app and see what they should do today. Natural language is the primary interface for creation and modification; structured UI is for execution (logging sets fast).

## Tech Stack
- **Platform:** iOS only (Swift / SwiftUI)
- **Storage:** Local on-device with iCloud sync
- **AI:** Claude API (BYOK — user provides their own Anthropic API key)
- **Health Data:** HealthKit integration (sleep, HRV, recovery metrics)
- **Auth:** None — single user, local-only

## Core Data Model

### Exercise
- Name (user-defined)
- Muscle group category
- Persisted to user's exercise library for reuse

### Workout Template
- Name (user-defined or AI-generated)
- Ordered list of exercises
- Each exercise has ordered sets, each set contains:
  - Weight
  - Reps
  - Rest time (seconds)
  - Tags: warmup, failure (optional)
- Templates serve as the AI's building blocks, not the user's primary interface

### Workout Log
- Actual recorded data per set
- Timestamp
- RPE / notes (optional)
- Stored locally with iCloud backup
- Historical logs are core AI context — fed into every recommendation

## Features

### 1. AI-Recommended Daily Workout (Hero Feature)
- Every day, the home screen shows "Today's Workout" generated by AI
- AI considers: workout history, muscle group recovery, progressive overload, user goals, HealthKit data (sleep, HRV, etc.)
- User can accept, modify via natural language, or request alternatives
- "I'm tired today" → AI adjusts intensity. "Skip legs" → AI reprograms.

### 2. Natural Language Workout Creation
- Users describe what they want in plain text
- "Build me a 4-day upper/lower split, hypertrophy focus, 60 min sessions"
- AI generates full program with templates
- Exercise creation also via NL: "add face pulls, 3x15 at 30 lbs, 1 min rest"
- Traditional forms exist as fallback, not primary path

### 3. Exercise Library
- User creates exercises from scratch or via natural language
- Assigns muscle group category
- Saved to personal exercise library for reuse

### 4. Workout Logging
- Start the AI-recommended workout (or any template)
- Log each set (weight, reps, tags)
- Rest timer auto-starts after logging a set
- Timer uses the rest time defined for that set
- Mid-workout AI adaptation: if you fail early or crush a set, AI adjusts remaining work

### 5. Rest Timer
- Per-set countdown timer
- Auto-starts when a set is logged
- When timer hits zero: push notification with sound (works when app is backgrounded)
- Visual countdown in-app

### 6. AI Workout Editing
- Text input field for natural language requests during or before a workout
- Examples: "shorten this to an hour", "add more chest volume", "make this a deload week"
- AI response shown as a diff in a slide-up modal
- User approves (checkmark) or dismisses
- Uses Claude API with user's own API key (BYOK)

### 7. Post-Workout Debrief
- AI-generated summary after finishing: volume vs. last session, PRs, fatigue flags
- "Bench volume up 8% from last week. OHP stalled 3 sessions — consider a deload."

### 8. Health Data Integration
- **HealthKit:** sleep duration/quality, HRV, resting heart rate, active energy
- **Whoop API:** recovery score, strain, sleep performance, HRV — ideal for training readiness
- AI uses this data to modulate workout intensity and recovery recommendations
- "Recovery at 42% — lighter session recommended today"
- "You slept 5 hours — reducing volume, keeping intensity moderate"

### 9. Data Storage
- All data stored on-device
- iCloud sync for backup/cross-device
- Historical workout data is core AI context

### 10. User Goals & Constraints (Persistent AI Context)
- Set during onboarding and editable anytime via natural language
- Training goals: hypertrophy, strength, endurance, general fitness
- Schedule: how many days/week, session duration
- Equipment constraints: "no bench", "home gym only", "dumbbells up to 50 lbs"
- Injuries/limitations: "bad left shoulder", "avoid heavy squats"
- Experience level: beginner, intermediate, advanced
- These persist and frame every AI decision — workout programming, exercise selection, progression
- User can update anytime: "I got a squat rack" or "switching focus to strength"

## Not in MVP
